Navigation

Pagination

Pagination breaks long lists or tables into navigable pages. The component combines previous/next chevrons, page-number tiles, and optional ellipsis to skip large gaps. The active page wears a blue border so it's always clear where you are in the set.

Variants

Arrows — use when:
  • You only need step-by-step "Previous / Next" navigation
  • Default has a 1px #CDD2D7 border with a dark gray icon
  • Disable the chevron at the start/end of the range to communicate the boundary
Numbers — use when:
  • Pages 1–N can be addressed directly
  • Active page wears a 1px #027CB1 border; default and hover have no border
  • Apply aria-current="page" to the active tile

Usage Guidelines

When to paginate
  • Use pagination when a dataset exceeds a single page of comfortable scrolling
  • Pair with results-per-page selection so users can choose a density that suits them
  • Show total results count when it helps users understand scope
Indicate current page
  • Use a clear visual treatment for the active page
  • Mirror it with aria-current="page" for screen reader users
Allow direct navigation
  • Surface the first and last page numbers so users can jump to extremes
  • Use truncation (…) to keep the control compact when there are many pages
Keep prev / next visible
  • Provide explicit Previous and Next controls — don't make users hunt for page changes
  • Disable Previous on the first page and Next on the last page (don't hide them)
Use for small lists
  • Don't show pagination when all results fit on a single page
  • Don't paginate fewer than ~10 items just to maintain a consistent layout
Hide page state
  • Don't omit the current-page indicator — users need to know where they are
  • Don't drop the total page count when more than one page exists
Replace search or filters
  • Don't use pagination as a substitute for search when users know what they want
  • Don't paginate datasets that would be better served by infinite scroll or virtualization
Shift layout on click
  • Don't allow the pagination control's width to change as page numbers grow
  • Don't reset scroll position to the top of the entire page — return focus to the top of the result list

Accessibility

  • ARIA: Wrap the group in <nav aria-label="Pagination">. Mark the active tile with aria-current="page" and give the chevrons accessible labels like "Previous page".
  • Keyboard: Every page tile and chevron must be reachable in tab order. Disabled chevrons should be skipped via the native disabled attribute.
  • Don't rely on color alone: The active tile uses both a blue border and the aria-current attribute, so screen readers and color-blind users still know which page is active.
  • Touch targets: Minimum 32×32 hit area meets WCAG; pad surrounding elements to keep adjacent tiles from triggering accidental taps.
