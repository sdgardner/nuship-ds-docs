Avatar

Avatars identify users through photos or initials. They appear in profiles, comments, tables, and navigation — providing visual recognition at any scale.

Usage Guidelines

Use for user identity
  • Profiles, comment authors, account menus
  • Data tables with user columns
Scale to context
  • xs/sm for dense tables or chips
  • md/lg for cards or threads
  • xl/xxl for profile pages
Don't use for brand or product logos
  • Use a dedicated logo component instead
Don't truncate initials beyond 2 characters
  • Use maximum 2-letter initials for readability at small sizes

Accessibility

  • Alt text: Images must have meaningful alt text or alt="" if purely decorative with a visible name nearby.
  • Screen readers: Initials avatars should have aria-label with the full user name so assistive technology announces the person's name rather than the initials.
  • Status badge: Include aria-label="Online" (or equivalent status wording) on the badge element so screen reader users are informed of the presence status.
  • Touch targets: Minimum touch target of 44×44px — wrap small avatars (xs, sm) in a larger hit area for interactive use to meet WCAG 2.5.5.
