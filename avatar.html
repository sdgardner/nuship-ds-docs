Display

Avatar

An Avatar represents a person or entity in the UI. It displays a profile photo when available, or falls back to the user's initials. An optional status badge communicates presence or activity state. Avatars scale across six sizes to suit different contexts — from inline mentions to profile headers.

Usage Guidelines

Photo quality
  • Ensure photos are square and face-centered before use
Right-size for context
  • Use the smallest size that still provides enough recognition
  • Keep sizes consistent within a list or group of avatars
Status badges
  • Use the badge to indicate online or active status
Reliable fallback
  • Use initials as a reliable fallback when no photo is available
Initials count
  • Don't use more than 2 initials — it won't fit
Badge misuse
  • Don't use the badge for non-presence states (errors, counts, etc.)
Mixed sizes
  • Don't mix sizes in the same context (e.g., a user list)
Inline sizing
  • Don't use xl or xxl for inline/body text contexts

Accessibility

  • Alt text: Images must have meaningful alt text or alt="" if purely decorative with a visible name nearby.
  • Screen readers: Initials avatars should have aria-label with the full user name so assistive technology announces the person's name rather than the initials.
  • Status badge: Include aria-label="Online" (or equivalent status wording) on the badge element so screen reader users are informed of the presence status.
  • Touch targets: Minimum touch target of 44×44px — wrap small avatars (xs, sm) in a larger hit area for interactive use to meet WCAG 2.5.5.
