Password Input

The Password Input is a specialised text field for entering sensitive credentials. It masks the value by default and provides a visibility toggle so users can reveal or conceal their password as needed.

Accessibility

  • Keyboard: Tab moves focus to the field. Tab again moves focus to the visibility toggle button. Enter or Space on the toggle reveals or conceals the password.
  • Screen readers: Use <input type="password"> for the field. Label the toggle button with aria-label="Show password" or "Hide password" and update it dynamically when toggled. Announce the state change with aria-live="polite".
  • Visual: Never remove the visibility toggle — it is a critical affordance for users with motor or cognitive impairments. Focus ring must be clearly visible (2px blue border + glow). Error state must not rely on colour alone — pair red border with the error icon and message text.
  • Touch targets: The visibility toggle icon area must be at least 44×44px on touch devices. Ensure adequate padding around the icon to prevent mis-taps with the input field.
