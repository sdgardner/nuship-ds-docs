Radio Button

The Radio Button component allows users to select exactly one option from a set of mutually exclusive choices. Radio buttons are used when only a single selection is permitted and all available options should be visible to the user.

Usage Guidelines

Mutually Exclusive Choices
  • When only ONE option can be selected from a list
  • Clear either/or decisions
  • Selection automatically deselects other options
Visible Options (Small Sets)
  • 2–7 options that should all be visible
  • When users benefit from seeing all choices at once
  • Options that need comparison before selection
Required Single Selections
  • Shipping method selection
  • Payment type selection
  • Account type or plan selection
  • Sort order or view preferences
Form Inputs with Clear Defaults
  • When one option should typically be pre-selected
  • Users need to make a conscious choice
  • Selection is required to proceed
Multiple Selections Allowed
  • When users can select more than one option (use checkboxes)
  • Multi-select filters or preferences
  • "Select all that apply" scenarios
Long Lists of Options
  • More than 7–8 options (use dropdown/select)
  • Options that require scrolling to view all
  • Dynamic or searchable option lists
Immediate Action Triggers
  • Actions that execute immediately upon selection (consider buttons)
  • Mode switching with instant feedback (use segmented control or tabs)
  • On/off states (use switch/toggle)
Optional Single Selections
  • When "none selected" is a valid state and must remain selectable
  • If users might want to deselect all options (use checkboxes or dropdown with "None" option)
Binary On/Off States
  • Simple enable/disable settings (use switch/toggle)
  • Single yes/no questions (use switch or checkbox)

Messaging

Radio Button Labels

Use clear, distinct labels for each radio option:

  • Keep labels concise but specific enough to differentiate options.
  • Use sentence case.
  • Place label to the right of the radio button.
  • Make the entire label clickable (not just the radio circle).
  • Ensure labels are mutually exclusive and don't overlap in meaning.

Example labels:

  • "Standard Shipping (5–7 days)"
  • "Express Shipping (2–3 days)"
  • "Overnight Shipping (Next day)"

Individual Radio Helper Text

Provide additional context for specific radio options when needed:

  • Use to clarify costs, timeframes, or consequences.
  • Keep brief (1 line preferred, 2 lines maximum).
  • Use lighter text color (color/text/secondary).
  • Place directly below the radio label.

When to use:

  • Option has additional costs or restrictions.
  • Selection has important consequences.
  • Clarification helps users make informed choice.

Example helper text:

  • "Additional $25 fee"
  • "Available for domestic shipments only"
  • "Recommended for time-sensitive deliveries"

Group Labels

Use group labels to frame the decision users need to make:

  • Phrase as a question or clear heading.
  • Include required indicator (*) when selection is mandatory.
  • Use sentence case.
  • Keep under 50 characters.

Example group labels:

  • "Shipping Method *"
  • "Payment Type"
  • "How would you like to be contacted?"
  • "Select your plan"

Group Helper Text

Provide context for the entire radio group when needed:

  • Explain what the selection affects.
  • Clarify any constraints or requirements.
  • Keep to 1–2 lines maximum.
  • Place below the group label, above the radio options.

Example group helper text:

  • "Choose one shipping option for this order"
  • "Changing your plan will take effect immediately"
  • "This selection cannot be changed after submission"

Error Messages

Show clear, actionable error messages for validation:

  • Display below the radio group.
  • Use red text (color/text/error).
  • Be specific about what needs to be fixed.
  • Appear after validation or attempted form submission.

Example error messages:

  • "Please select a shipping method"
  • "Payment type is required"
  • "You must choose one option to continue"

Accessibility

Keyboard Shortcut

  • Tab — navigate to radio group (focuses first/selected radio).
  • Arrow Up/Down (or Left/Right) — navigate between options in the group.
  • Space — select the focused radio (if not already selected).
  • Shift + Tab — navigate to previous form element.

Screen Readers

  • Use proper <input type="radio"> markup with the same name attribute for grouped radios.
  • Include aria-label or associate visible labels with for/id.
  • Group related radios with <fieldset> and <legend>.
  • Announce which option is selected and the total number of options in the group.
  • Indicate required groups with aria-required="true".

Visual

  • Focus indicator must be clearly visible (2px blue ring).
  • Minimum 3:1 contrast ratio for radio against background.
  • Labels must meet WCAG AA contrast (4.5:1).
  • Selected state must be distinguishable without relying on color alone.

Touch Targets

  • Radio + label hit area minimum 44×44px.
  • Ensure adequate spacing between radio options (at least 8px).
  • Make entire label clickable, not just the radio circle.

Groups

  • Always use one radio group per set of related options.
  • Ensure all radios in a group have the same name attribute.
  • Only one radio in a group can be selected at a time.
  • Provide clear group labels to establish context.

Related Components

Checkbox
Use a Checkbox when the user can select multiple options. Use Radio when exactly one choice is allowed.
Dropdown
Use a Dropdown when the set of options is long (>5) and a radio group would take too much vertical space.
Switch
Use a Switch for a binary on/off toggle. Use Radio when there are three or more mutually exclusive choices.
