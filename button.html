Actions

Button

Buttons trigger immediate actions or navigate the user to another page. They are available in multiple styles to communicate hierarchy and intent, and in three sizes to fit different layout contexts.

Developed Figma

Anatomy

1 3 2 4
1Container
2Label
3Leading Icon (optional)
4Trailing Icon (optional)

Variants

Use when:
  • Triggering the single most important action on a page or dialog
  • Confirming or submitting a task flow
  • Only one Primary button should appear per view
Token group:
  • color/action/primary
Use when:
  • Offering a supporting action alongside a Primary button
  • The action is important but not the critical path
  • Multiple equal-priority actions are available
Token group:
  • color/action/secondary
Use when:
  • Supplementary actions need a visible affordance without overwhelming a Primary
  • Use in toolbars or dense interfaces where a solid fill would be too heavy
  • Stacking alongside solid buttons for optional or alternative paths
Token group:
  • color/border/button-tertiary
Use when:
  • Ancillary actions should recede into the interface without visual weight
  • Embedding in cards, list rows, or inline alongside other content
  • Providing optional controls that shouldn't compete with primary actions
Token group:
  • color/bg/ghost
Use when:
  • The primary action is irreversible — Delete, Remove All, Clear
  • The destructive action is the main focus of a confirmation dialog
  • High visual prominence is needed to signal risk
Token group:
  • color/action/destructive
Use when:
  • A destructive action is available alongside neutral or positive actions
  • Deletion exists but is not the recommended or primary path
  • Reduced visual weight is appropriate for the context
Token group:
  • color/border/button-destructive
Use when:
  • A low-emphasis destructive option should not draw immediate attention
  • Embedded in dense interfaces like table rows or inline editors
  • The destructive path is rare and should feel accessible but unobtrusive
Token group:
  • color/text/button-destructive

Properties

PropertyTypeDefaultDescription
label string — Text displayed inside the button. Not shown in icon-only content.
style Primary | Secondary | Tertiary | Ghost | Destructive Primary | Destructive Tertiary | Destructive Ghost Primary Visual and semantic treatment. Communicates hierarchy and destructive intent.
size Large | Medium | Small Large Controls padding and icon size. Large = 16px/24px padding; Medium = 12px/16px; Small = 4px/8px.
content Text + Icon | Icon Only Text + Icon Icon-only buttons use square padding and require an accessible aria-label.
state Enable | Hover | Pressed | Focus | Disabled | Skeleton Enable Interactive state. Disabled buttons remain in the DOM but are non-interactive.
