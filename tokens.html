The three pillars

Building a Better
Foundation.

Every decision in nuShip rolls up to three principles. If a token, component, or pattern doesn't serve one of them, it doesn't ship.

01

Clarity

Every token names a role, not a value. Designers and engineers reach for the same word when they mean the same thing.

02

Consistency

One source of truth from Figma to code. Color, spacing, radius, and type all resolve to the same primitives — no drift between surfaces.

03

Scalability

New themes, new modes, and new brands plug into the layer system without rewriting components. Foundation first, features after.

The color story

A palette with room to breathe.

nuShip ships an expanded eleven-step gray ramp — from a pure white shell to a near-black ink. That depth gives us three surface layers stacked clearly without relying on heavy shadow.

Dark mode, by design

The dark side
of the moon.

Dark mode isn't a re-skin — it's a parallel set of semantic values that maps onto the same component contracts. Components don't know which theme is active.

01

System-first

Every component reads from semantic tokens. Theme switching is one CSS variable swap away — no per-component overrides.

02

No rework

Designers don't redraw screens for dark mode. The token map handles the translation; the layout stays identical.

03

Consistent feel

Status, action, and surface roles keep their meaning across modes. Error still reads as error, just calibrated for the new background.

How tokens resolve

Speaking the same language.

A button's background isn't a hex. It's a chain — a component token pointing at a semantic role, pointing at a base scale, pointing at a raw primitive. Change one link and the whole system updates in lockstep.
