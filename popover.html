Popover

A Popover is a non-modal floating container anchored to a trigger element. It surfaces contextual information, actions, or a short confirmation prompt without navigating away or blocking the full interface.

Accessibility

  • Role: Add role="dialog" to the popover container when it contains interactive elements (actions, confirm buttons). For info-only content, role="tooltip" or role="region" is appropriate. Always pair with aria-label or aria-labelledby.
  • Focus management: When the popover opens and contains interactive elements, move focus to the first focusable element inside it. When it closes, return focus to the trigger element that opened it.
  • Keyboard interaction: Escape must close the popover from any point within it. If the popover is a dialog, trap focus inside it while it is open so Tab does not cycle into background content.
  • Trigger state: Use aria-expanded="true/false" on the trigger element so screen readers announce whether the popover is open or closed.
  • Colour contrast: Ensure body text inside the popover meets WCAG AA (4.5:1 for small text). The danger action text colour #A12F2F (light) and #E36868 (dark) should be verified against the container background on each theme.

Related Components

Modal
Use a Modal when the content or action requires the user's full focus and must block the rest of the interface.
Dropdown
Use a Dropdown when selecting a value from a predefined list. Popovers are for contextual actions, not selection.
Tooltip
Use a Tooltip for a single short label on hover. Popovers are richer and always opened by click or focus.
