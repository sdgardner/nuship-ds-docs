Date Picker

The Date Picker allows users to select a single date, date range, or date and time. It opens a calendar popover from an input trigger, supporting keyboard navigation and accessible date selection across scheduling, filtering, and booking workflows.

Properties

PropertyTypeDefaultDescription
type "single" | "range" | "datetime" "single" Picker variant controlling layout and selection behavior.
label string — Label text displayed above the input trigger.
placeholder string "Select a date" Placeholder text shown when no value is selected.
value string — Selected date string for single and datetime variants.
startDate string — Range start date (range variant only).
endDate string — Range end date (range variant only).
disabled boolean false Disables the input trigger and prevents the calendar from opening.
minDate string — Earliest selectable date; days before this are rendered disabled.
maxDate string — Latest selectable date; days after this are rendered disabled.

Accessibility

Keyboard

  • Arrow keys navigate days within the calendar grid.
  • Enter / Space selects the focused day.
  • Escape closes the calendar popover and returns focus to the trigger.
  • Tab moves focus between distinct UI regions: trigger, calendar header, day grid, and footer buttons.

Screen Readers

  • The calendar grid uses aria-label with the current month and year so the context is announced when focus enters.
  • Each day button has an aria-label with the full date (e.g. "March 18, 2026") and an aria-pressed or aria-selected attribute to communicate the selected state.
  • Disabled days use aria-disabled="true" and are excluded from the tab order.

Focus Management

  • When the calendar opens, focus moves into the calendar — typically to the selected day or to the current date if no selection exists.
  • When the calendar closes via Apply, Clear, or Escape, focus returns to the input trigger that opened it.

Touch

  • Each day cell maintains a minimum 44×44px touch target to meet WCAG 2.5.5 target size guidance.
  • The calendar should be dismissible by tapping outside the popover boundary without committing a selection.
