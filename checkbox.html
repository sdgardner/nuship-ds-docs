Checkbox

The Checkbox component allows users to select one or more options from a list. Checkboxes are used for non-exclusive choices where users can select multiple items, and can also represent a parent-child relationship through the indeterminate state.

Usage Guidelines

Use checkboxes for multiple selections
  • Selecting multiple items from a list
  • Filters where users can apply several criteria at once
  • Multi-select forms where 0, 1, or many options are valid
Single toggle options
  • On/off preferences or settings
  • Optional features that can be enabled/disabled
  • Agreement to terms or policies
Task or item lists
  • To-do lists or task completion tracking
  • Selection of items for batch actions
  • Item selection in data tables
Hierarchal selections
  • Parent-child relationships with "Select All" functionality
  • Nested categories with indeterminate states
  • Group selections with sub-options
Don't use for mutually exclusive choices
  • When only one option can be selected — use Radio Buttons
  • Binary either/or decisions between two options
  • Single selection from a list
Don't use for immediate action triggers
  • Actions that take effect immediately — use a Switch
  • Settings with instant visual feedback
  • States that don't require form submission
Don't use for navigation or mode switching
  • Switching between views or modes — use Tabs or segmented controls
  • Filter buttons that toggle on/off — use Toggle Buttons
Don't use for long lists without search
  • More than 10–15 options without filtering — use a multi-select dropdown
  • Very long lists that require scrolling — consider dropdown with search

Accessibility

  • Keyboard: Tab navigates between checkboxes. Space toggles on/off. Shift + Tab navigates backwards.
  • Screen readers: Use proper <input type="checkbox"> markup. Include aria-label or associate visible labels with for/id. For indeterminate state, set aria-checked="mixed". Group related checkboxes with <fieldset> and <legend>.
  • Visual: Focus indicator must be clearly visible (2px blue ring). Minimum 3:1 contrast ratio for checkbox against background. Labels must meet WCAG AA contrast (4.5:1). Don't rely on color alone to convey checked state.
  • Touch targets: Checkbox + label hit area minimum 44×44px. Ensure adequate spacing between checkboxes (at least 8px). Make entire label clickable, not just the checkbox square.
