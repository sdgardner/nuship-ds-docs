Checkbox

A checkbox allows users to select one or more options from a set. Each checkbox operates independently — checking one does not affect others. It supports optional description text and an error state for validation feedback.

Usage Guidelines

Multiple independent choices
  • Use when users can select more than one option at a time
  • Each checkbox should represent a standalone, independent choice
  • Use for preferences, filters, and feature toggles
Indeterminate for parent selection
  • Use indeterminate state when a parent checkbox controls a group
  • Show indeterminate when only some children are checked
  • Selecting the parent should check all children
Don't use for single exclusive choice
  • Don't use checkboxes when only one option can be selected — use Radio Buttons
  • Avoid using a single checkbox as a binary yes/no decision — use a Switch
  • Don't use checkboxes for navigation or triggering actions
Don't overload with options
  • Avoid long lists of checkboxes without grouping or hierarchy
  • Don't mix checked and disabled states without clear visual distinction
  • Avoid hiding the label — every checkbox needs a visible label

Accessibility

  • ARIA role: Use role="checkbox" with aria-checked="true", "false", or "mixed" for indeterminate.
  • Keyboard: Reachable via Tab. Toggle with Space. Does not use arrow keys — each checkbox is independently focusable.
  • Labels: Every checkbox must have a visible label. Use aria-label or aria-labelledby if the visual label is separate.
  • Error state: Associate error messages using aria-describedby pointing to the error text element.
  • Color contrast: Border and fill colors meet WCAG AA. Focus ring provides a visible 4px outline for keyboard users.
