Search

The Search field lets users filter or find records by typing a query. It appears in two sizes — an expanded bar with placeholder text and a minimized icon-only trigger — and supports placeholder, active, hover, focus, and disabled states.

Expanded states:
  • Default — idle field with placeholder text and clear button
  • Hover — border darkens to #6B7280 on pointer enter
  • Focus — 1px solid ring + 4px translucent glow
  • Active — user has typed a query, text uses primary colour
  • Disabled — transparent background, muted border and text
Use when:
  • Screen space is limited (e.g., toolbar, mobile nav)
  • Clicking the icon expands the full search field
  • Shows only the magnifying glass icon — no placeholder or clear button

Accessibility

  • Role: Use role="search" on the containing landmark and a native <input type="search"> inside it. This lets screen readers identify the search region and enables the browser's built-in clear affordance on some platforms.
  • Label: Add an aria-label or a visually-hidden <label> that describes the scope (e.g., "Search booked shipments"). Placeholder text alone is not an accessible name.
  • Clear button: Give the clear icon an aria-label="Clear search". When pressed, move focus back to the input so the user can type a new query immediately.
  • Keyboard: The field should receive focus with Tab. Escape should clear the value (if non-empty) or blur the field (if already empty). The clear button should be reachable via Tab or as a secondary action on Escape.
