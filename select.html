Select

The Select component lets users choose a single option from a list presented in a native browser dropdown. It includes a label, placeholder, helper text, and supports hover, focus, open, error, and disabled states.

States:
  • Default — idle field with placeholder and helper text
  • Hover — border darkens to #6B7280
  • Focus — 1px blue ring + 4px glow around the field
  • Open — chevron flips up, native browser menu appears
  • Error — red border + error icon with message
  • Disabled — transparent background, muted text and border

Accessibility

  • Native element: Use a native <select> element so the browser provides keyboard navigation, ARIA roles, and mobile-optimized pickers automatically. Avoid custom dropdown implementations unless the native element cannot meet requirements.
  • Label association: Associate the visible label with the <select> via a <label for="..."> element. Screen readers will announce the label when the field receives focus.
  • Error messaging: Link the error message to the field with aria-describedby and add aria-invalid="true" in the error state. Screen readers will announce the error when the user focuses the field.
  • Keyboard: Space or Enter opens the native dropdown. Arrow Up/Down moves between options. Escape closes the dropdown without changing the value.

Related Components

Dropdown
Use a Dropdown for custom-styled option lists with tags, icons, or multi-select. Select uses the browser's native menu.
Radio Button
Use Radio Buttons when there are 2–5 options and all choices should be visible at once without opening a menu.
Search
Use Search when the user needs to filter a large set by typing. Use Select when the options are predefined and short.
