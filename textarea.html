Text Area

The Text Area is a multi-line text input for longer freeform content such as notes, instructions, or descriptions. It supports a label, optional character counter, helper text, and multiple interaction states including error and read-only.

States
  • Default — empty field waiting for input
  • Hover — cursor over the field, border darkens
  • Focus — field is focused, teal ring appears
  • Active — user is typing, value is present
  • Disabled — field is not interactive
  • Error — validation failed, red border and message
  • Error Limit — character limit reached
  • Read Only — value shown but not editable

Usage Guidelines

Use for multi-line input
  • Use when the user needs to enter more than a single line of text
  • Good for notes, instructions, comments, and descriptions
  • Set a maxLength and show the counter for inputs with known limits
Provide helpful context
  • Always include a label — never rely on placeholder text alone
  • Use helper text to set expectations (format, limits, examples)
  • Show error messages inline, directly below the field
Don't use for short inputs
  • Don't use a Text Area for single-word or single-line inputs — use Text Input instead
  • Avoid using it for structured data like dates, phone numbers, or amounts
Don't leave errors unresolved
  • Don't show an error state without a clear, actionable error message
  • Avoid triggering errors before the user has finished typing

Accessibility

  • Label association: Always connect the label to the textarea using for / id pairing or aria-labelledby. Never rely on placeholder text alone.
  • Character counter: Announce counter changes with aria-live="polite" so screen reader users hear updates as they type.
  • Error messages: Link the error message to the field via aria-describedby and set aria-invalid="true" on the textarea when in an error state.
  • Keyboard: The field must be reachable via Tab. Avoid trapping focus — Tab within a textarea should insert a tab character only if explicitly required.
  • Read only: Use readonly attribute rather than disabled when the value is meaningful but not editable, so it remains selectable and copyable.
