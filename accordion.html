Accordion

An accordion is a vertically stacked set of expandable sections. Each section contains a trigger that reveals or hides associated content. Accordions help reduce visual clutter by progressively disclosing information on demand.

defaultOpen boolean false Sets the initial open state (uncontrolled mode).
disabled boolean false Prevents the accordion from being toggled.
onChange function — Callback fired when the expanded state changes. Receives the new boolean value.

Usage

Progressive disclosure of secondary content
  • When detailed information is relevant but not needed at first glance
  • Content like policies, reviews, or legal text that some users want and others don't
  • When reducing visual complexity helps users focus on primary tasks
FAQ and help content
  • Contexts where users have a specific question they're scanning to find
  • When multiple independent questions or topics need to coexist on one screen
  • Help sections, onboarding explanations, and support flows
Grouped detail sections on detail screens
  • Carrier or shipper profile pages where multiple content categories exist
  • Listing detail screens with supplemental information (reviews, insurance, terms)
  • When users may want to compare content across collapsible sections
Long-form content management
  • When a page has too much content to show at once without overwhelming the user
  • Settings or configuration panels where not all options are always relevant
Task critical primary information
  • Pickup and delivery addresses — users need these visible to complete a booking
  • Pricing or quote totals that directly drive a decision
  • Required fields or actions in a form or checkout flow
Single short content blocks
  • When there's only one section — just show the content inline
  • When the content inside is just 1–2 sentences — the overhead isn't worth it
  • Avoid creating accordion "theater" where the expand adds no real value
Sequential or step-based flows
  • Multi-step forms — use stepper or wizard patterns instead
  • Onboarding flows where order of completion matters
  • When the user needs to read everything before taking action
Navigation
  • Don't use accordions as a substitute for page navigation or tabs
  • Avoid nesting interactive elements like buttons or forms inside panels
  • Primary CTAs should never be hidden behind an accordion

Accessibility

Keyboard Shortcut

  • The accordion trigger must be reachable and activatable by keyboard — Tab to focus, Enter or Space to toggle open/closed.
  • All accordion triggers on a screen must be included in the natural tab order.
  • Focus must not be trapped inside the accordion panel when it is expanded.
  • When an accordion is disabled, its trigger must not receive keyboard focus.

Screen Readers

  • The accordion trigger must have role="button" or be a native <button> element with a visible label.
  • Use aria-expanded="true" / aria-expanded="false" on the trigger to communicate open/closed state.
  • The content panel should be associated with the trigger via aria-controls pointing to the panel's ID.
  • Collapsed panels should use aria-hidden="true" or hidden so screen readers skip their content.
  • When a section is disabled, use aria-disabled="true" on the trigger element.
  • Trigger labels must be descriptive enough to make sense out of context — avoid generic labels like "More" or "Details".

Focus Management

  • When an accordion is toggled open, focus remains on the trigger — it should not jump into the content.
  • When an accordion is toggled closed, focus should return to the trigger that initiated the action.
  • If expanding an accordion causes content to shift significantly, ensure the layout change does not disorient screen reader users.

Visual

  • The focus ring using color/border/focus-ring must always be visible — never suppress it with outline: none.
  • The teal trigger label color (color/text/button-tertiary/default) must meet WCAG AA contrast (4.5:1) against the white background.
  • The disabled state must communicate unavailability through more than color alone — reduced opacity reinforces this alongside the muted token.
  • The chevron icon alone does not convey the open/closed state to screen readers — aria-expanded is required as the semantic signal.
