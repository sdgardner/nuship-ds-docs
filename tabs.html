Navigation

Tabs

Tabs (in UI) are a control that lets people switch between multiple panels of content in the same space. Only one panel is shown at a time; choosing a tab shows its panel and hides the others.

Usage Guidelines

Content & structure
  • Keep labels short (1–2 words)
  • Use about 2–7 tabs; for more options, use another pattern (e.g. dropdown, nav list)
  • Use either icon-only tabs or text-with-icon tabs in a set — keep the pattern consistent within one tab list
Selection & visibility
  • Always show one selected tab
  • Make the selected state obvious (underline, background, or border)
  • Match the indicator to the tab (full or content width)
Visual design
  • Use consistent spacing and alignment between label, icon, and badge
  • Use badges only when they add value; hide when a tab is disabled
Content & structure
  • Don't use tabs for a single item
  • Don't nest tabs inside tabs
  • Avoid long or wrapping labels
  • Don't mix icon-only and text-with-icon tabs in the same row
Selection & visibility
  • Don't have a tab go to the same panel
States and behavior
  • Don't skip a visible focus state
  • Don't show error state without a clear error message
  • Don't make disabled or read-only look editable; reduce contrast and remove resize when disabled
  • Don't switch panel on hover only
  • Don't use tabs when the items navigate to a new page (use links/nav instead)

Accessibility

  • Keyboard: Arrow Left/Right (horizontal) or Arrow Up/Down (vertical) move focus between tabs and select the focused one. Home jumps to the first tab, End to the last.
  • ARIA: Use role="tablist" on the container, role="tab" on each tab item, and role="tabpanel" on the associated content panel. Set aria-selected="true" on the active tab and aria-selected="false" on all others.
  • Focus ring: The 1px solid ring + 4px glow is visible in both themes. Ensure focus is never suppressed on keyboard navigation. Disabled tabs should be removed from the focus order.
  • Colour contrast: Active blue text (#027CB1 light) on white background meets WCAG AA at 4.6:1. Badge text (white on blue fill) also passes. In dark mode, verify lighter blue (#5ABFE0) against dark backgrounds.

Related Components

Segmented Control
Use Segmented Control for switching a view mode within one panel. Tabs are for separate content panels.
Accordion
Use Accordion when content sections can be expanded/collapsed independently rather than switching between them.
Menu
Use Menu for navigation with many items or nested hierarchies. Tabs work best with 2-7 items.
