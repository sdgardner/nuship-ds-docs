Number Input

The Number Input allows users to enter or adjust a numeric value using increment and decrement controls. It is ideal for quantities, counts, or any scenario where a precise integer value is needed within a defined range.

Use when:
  • The user needs to input or adjust a precise numeric value
  • Value changes are incremental (e.g. quantity, passengers, items)
  • Direct keyboard entry alongside stepper controls is acceptable
Visual changes by state:
  • Hover — stronger border colour
  • Focus — blue border with glow ring
  • Error — red border, helper text replaced by error message
  • Disabled — faded border, icons and text all appear muted

Accessibility

  • Keyboard: Tab moves focus into the field. ↑ / ↓ arrow keys increment and decrement the value. Home / End jump to the minimum and maximum values when defined.
  • Screen readers: Use role="spinbutton" on the input. Set aria-valuenow, aria-valuemin, and aria-valuemax. Associate the label with aria-labelledby or for/id. Announce live value changes with aria-live="polite".
  • Visual: Focus indicator must be clearly visible (blue border + glow ring). Ensure value text meets WCAG AA contrast (4.5:1). Disabled state must still be perceivable — don't rely on opacity alone.
  • Touch targets: Decrement and increment buttons must each be at least 44×44px on touch devices. Provide sufficient spacing between the two controls to prevent mis-taps.
