Actions

Tag

Pill-shaped chips for filters, selections, and dropdown triggers. Tags toggle between Default and Active to show selection, optionally render a leading icon (e.g. Plus to add) and a trailing dismiss X, and come in two sizes — 40px Default and 24px Compact.

Not Developed Figma

Anatomy

1 2 3 4
1Container (1px border, 100px radius)
2Leading icon (optional)
3Label (Bold when active)
4Dismiss / chevron icon

Variants

Default — use when:
  • The tag represents an unselected option in a filter or chip group
  • Surface uses color/surface/layer-01 with a 1px gray border
  • Optional leading plus icon signals "add this tag"
Active — use when:
  • The tag is currently selected or applied as a filter
  • Surface flips to color/status/info/bg (#E5EEF5) with a blue border + bold blue label
  • Click the X to remove the tag from the active set
Dropdown — use when:
  • The tag opens a menu of related options (filter facets, multi-select)
  • Trailing chevron replaces the dismiss icon
  • Active state shows when at least one option in the dropdown is selected

Usage Guidelines

Filter and selection
  • Use tags to represent active filters, multi-select chips, or keyword selections
  • Provide a close icon so users can remove a tag inline
  • Group related tags in a single horizontal row with consistent spacing
Keep labels concise
  • Use 1–2 short words per tag so the row stays scannable
  • Use sentence case unless the value is a proper noun
Keyboard accessibility
  • Make the remove button focusable and activatable with Enter or Space
  • Move focus to the next tag (or back to the input) when a tag is removed
Pair with input
  • When tags are user-generated, place them next to the input that created them
  • Animate insert and remove subtly so the row change is noticed without being jarring
Use as a primary action
  • Don't use tags as buttons or navigation links — use a Link or Button instead
  • Don't trigger destructive actions from a tag's close icon without a clear undo
Confuse with Badge
  • Don't use a Tag when the value is non-removable status — use a Badge instead
  • Don't mix Badge and Tag styles in the same row; the visual contract should be obvious
Overflow uncontrollably
  • Don't allow tag labels to wrap mid-word or stretch wider than the surrounding container
  • Don't show more than ~7 tags inline — collapse extras behind a "+N more" affordance
Hide the remove control
  • Don't hide the close icon behind hover only — it must be visible and focusable for keyboard and touch users
  • Don't rely on a small hit area; pad the close icon to a minimum 24×24 px target

Accessibility

  • ARIA: Dismissible filter tags should expose aria-pressed for the toggle state. Dropdown tags need aria-haspopup="listbox" and aria-expanded.
  • Two interactive zones: When the tag has both a body action and a trailing X, render them as separate <button> elements so each is independently focusable. Don't nest buttons.
  • Keyboard: Tag and its X both reachable via Tab. Enter / Space activate. For filter chips, Backspace when focused on the tag itself can also remove it.
  • Touch targets: Compact (24px) is below the 32×32 mobile minimum — only use Compact in pointer-input contexts (dense desktop filter rails) or pad the surrounding hit area.
