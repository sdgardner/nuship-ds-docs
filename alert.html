Alert

Alerts communicate important messages to users — errors, warnings, confirmations, or contextual information. They appear inline within a page or section and can be dismissed when no longer needed.

Something went wrong. Please try again.
Success alert
Your changes have been saved.
Info alert

Usage Guidelines

Use for system feedback
  • Show alerts after user actions like submitting a form
  • Use to communicate system status changes
  • Place alerts close to the content they relate to
Match variant to message
  • Use error for failed actions or validation issues
  • Use success for confirmations and completed tasks
  • Use info for neutral guidance or announcements
Don't overuse alerts
  • Don't show multiple alerts simultaneously on the same page
  • Avoid using alerts for non-actionable information
  • Don't use alerts as persistent decorative elements
Don't block the page
  • Don't use alerts for critical decisions — use a Modal instead
  • Don't use inline alerts for temporary notifications — use Toast
  • Avoid placing alerts in areas users rarely see

Accessibility

  • ARIA role: Use role="alert" for error/warning. Use role="status" for success/info.
  • Keyboard: Dismiss button reachable via Tab, activated with Enter or Space.
  • Color contrast: All text/icon pairs pass WCAG AA (4.5:1). Icon always pairs with color — never color alone.
  • Screen readers: Add aria-live="polite" for dynamically injected alerts so they're announced automatically.
