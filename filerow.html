File Row

The File Row displays a single file's upload status within a file upload flow. It communicates per-file progress, completion, and failure states, and allows users to remove files from the upload queue. It uses the Progress Bar component internally to visualize upload progress.

Usage Guidelines

File Upload Management
  • Displaying individual files queued for or actively uploading
  • Showing upload progress, completion, or failure per file
  • Allowing users to cancel or remove files from the upload queue
Upload State Communication
  • Queued — file is pending upload, no progress yet
  • Uploading — progress bar actively filling with percentage shown
  • Complete — green bar at 100%, success icon confirms upload
  • Failed — red bar with error message explaining why (e.g. "File too large — max 10 MB")
Contextual File Metadata
  • Showing file name, file size, and a thumbnail or file type icon together in one row
Non-Upload File Lists
  • Browsing or selecting files from a library — use a file picker or list component instead
Completed File History
  • Displaying a static list of previously uploaded files without upload state — use a plain list or table row
Bulk Upload Summary
  • Summarizing totals across multiple files — show individual File Rows for each file, not a single aggregated row

Accessibility

Keyboard Shortcut

  • Tab moves focus to the remove/cancel button.
  • Enter / Space activates the remove/cancel button.
  • No navigation within the row itself is required.

Screen Readers

  • The file name, status, and file size should be grouped and read as a unit.
  • Status label changes (Queued → Uploading → Complete / Failed) should be announced via aria-live="polite" for non-critical updates and aria-live="assertive" for errors.
  • The remove button should have a descriptive aria-label (e.g. aria-label="Remove photo_001.jpg") — not just an icon with no label.
  • Error messages must be programmatically associated with the file row using aria-describedby.
  • The progress bar within the row should follow the Progress Bar accessibility requirements.

Visual

  • Status must be communicated with both color and a label or icon — never color alone.
  • Error state must be clearly distinguishable from success and uploading states.
  • The remove button icon must have sufficient contrast against the background.

Touch Targets

  • Remove/cancel button must meet minimum 44×44px tap target.
  • Ensure adequate padding around the button so it is not accidentally triggered.
