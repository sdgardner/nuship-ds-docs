Drop-down

The Dropdown component allows users to choose one or more options from a dropdown list. It supports single and multi-selection with tag chips, keyboard navigation, and accessible option discovery for forms, filters, and configuration panels.

Accessibility

Keyboard Navigation

  • Tab moves focus to and from the Select trigger.
  • Enter / Space opens or closes the dropdown list.
  • Arrow keys navigate between options in the open list.
  • Escape closes the dropdown and returns focus to the trigger.

Screen Reader

  • The trigger uses role="combobox" so assistive technologies announce it as a composite input.
  • aria-expanded on the trigger reflects the open or closed state of the dropdown list.
  • aria-label on the field provides a descriptive name when no visible label is present.

Error Announcement

  • aria-describedby links the trigger to the error message element so the error is announced immediately when focus enters the field in an invalid state.
  • The error message element uses role="alert" or aria-live="polite" to surface validation feedback to screen reader users without requiring focus movement.
