Progress Bar

The Progress Bar communicates the status and completion of an ongoing operation. It appears in four statuses — default (idle), uploading, error, and success — and two sizes to suit different layout contexts.

Variants

Default — 6px
  • Use in standard layouts where vertical space is available
  • Preferred size for file upload and data transfer contexts
States:
  • Default — empty track, operation not yet started
  • Uploading — blue fill at 62% showing active progress
  • Error — red fill at same width showing where it failed
  • Success — green fill spanning full track width
Compact — 4px
  • Use in space-constrained contexts like table rows or sidebars
  • All four statuses behave identically to Default size
States:
  • Default — empty track, operation not yet started
  • Uploading — blue fill at 62% showing active progress
  • Error — red fill at same width showing where it failed
  • Success — green fill spanning full track width

Accessibility

  • Role: Use role="progressbar" on the track element. Set aria-valuenow, aria-valuemin="0", and aria-valuemax="100" to communicate the current percentage to screen readers.
  • Label: Always provide an aria-label or aria-labelledby that describes what is progressing (e.g., aria-label="Uploading document"). Without a label, screen readers cannot contextualise the bar.
  • Status announcements: When the status changes to Error or Success, announce the outcome with a live region: aria-live="polite" or role="status" on a nearby text element.
  • Colour independence: Do not rely on fill colour alone to communicate status. Pair the bar with an adjacent label, icon, or status message (e.g., "Upload failed" or "Upload complete").
