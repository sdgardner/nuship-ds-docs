Delete
Use when:
  • The action is irreversible — delete, remove, or permanently clear data
  • Label and icon render in --err-tx at rest; hover fills the row with --action-destructive and white text
  • Always separate from other actions with a divider and place at the bottom of the menu

Properties

PropertyTypeDefaultDescription
type "item" | "submenu" | "selected" | "destructive" "item" Determines the visual treatment and trailing element of the menu item.
state "default" | "hover" | "disabled" "default" Interaction state. Disabled removes pointer events and shifts all text to tertiary.
label string — The text content of the menu item. Keep concise — 1 to 3 words is ideal.
showIcon boolean true Toggles the leading 16×16 Lucide icon. When shown, all items in a group should include an icon for alignment.
keyboardShortcut string — Optional keyboard shortcut displayed on the trailing side (e.g. ⌘C, ⌃Z). Only applies to type "item".

Accessibility

Keyboard navigation

KeyAction
↑ / ↓Move focus between menu items
Enter / SpaceActivate the focused item
→Open a submenu when focus is on a submenu item
←Close current submenu and return focus to parent item
EscClose the menu and return focus to the trigger element
TabCloses the menu; Tab focus moves to next interactive element

ARIA attributes

  • The menu container uses role="menu" or <nav> with a descriptive aria-label
  • Each item uses role="menuitem" — or role="menuitemcheckbox" for toggleable selected items
  • Disabled items include aria-disabled="true" rather than the HTML disabled attribute (keeps them focusable for screen readers)
  • Submenu items include aria-haspopup="menu" and aria-expanded="true/false"
  • The trigger button that opens the menu should have aria-haspopup="menu" and aria-expanded
  • When the menu closes, focus returns to the trigger element
