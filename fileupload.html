File Upload

The File Upload Row component displays the progress and status of an individual file being uploaded. It communicates state through a progress bar, status text, and colour-coded iconography — covering queued, uploading, complete, and failed conditions.

Upload row states
  • One row per file — stack multiple rows for batch uploads
  • The row persists after success or failure so users can review the outcome
Visual changes by state:
  • Default — empty progress bar track; status "Queued"; blue icon tile border
  • Uploading — partial fill (#027CB1); percentage shown; status "Uploading..."
  • Success — full green fill; 100% shown; "Complete" in green; circle-check icon
  • Error — partial red fill; "Failed" label; error message; circle-x icon

Properties

Property Type Default Description
status 'default' | 'uploading' | 'success' | 'error' 'default' Controls the visual state of the row.
filename string — The name of the file being uploaded.
fileSize string — Formatted file size string shown at bottom right (e.g. "3.1 MB").
progress number 0 Upload progress value from 0 to 100. Drives the progress bar fill width.
onRemove function — Callback fired when the remove (circle-x) button is clicked.
showFileSize boolean true Toggles visibility of the file size label.
showStatusText boolean true Toggles visibility of the status text below the progress bar.
showIcon boolean true Toggles visibility of the file type icon tile.

Accessibility

Keyboard Navigation

  • Tab moves focus to the remove button on each upload row.
  • Enter / Space activates the remove button, cancelling or dismissing the file.

Screen Reader

  • The progress bar uses role="progressbar" with aria-valuenow, aria-valuemin="0", and aria-valuemax="100" to communicate upload progress.
  • The filename and current status should be combined in an aria-label on the row container so assistive technologies announce both together (e.g. "photo_001.jpg — Uploading, 54%").
  • The remove button uses aria-label="Remove photo_001.jpg" to provide a descriptive accessible name.

Status Announcements

  • State transitions (success, error) should be announced via aria-live="polite" on the status text element so screen reader users are notified without losing focus.
  • Error messages (e.g. "File too large — max 10 MB") should be persistent and not auto-dismissed, ensuring users have time to read the message.
