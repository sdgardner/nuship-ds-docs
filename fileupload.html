File Upload

The File Upload component provides a drop zone for dragging and dropping files, or browsing the file system. It manages the full upload lifecycle — from idle drop zone to per-file progress rows — with clear visual states for uploading, complete, and error conditions.

With files
  • Zone switches from the centred drop area to a stacked file list once files are selected
  • Each file row shows its own status; the zone border reflects the overall batch status
Visual changes by state:
  • Uploading — solid blue border; rows show mixed per-file states; warning footer text; Add More button
  • Complete — solid green border; all rows green; "N files uploaded" summary; Add More button
  • Error — solid red border; failed rows shown in red; error count summary; Retry Failed button

Properties

Property Type Default Description
state 'default' | 'hover' | 'dragging' | 'uploading' | 'complete' | 'error' 'default' Controls the overall visual state of the drop zone.
files FileRow[] [] Array of file objects. Each item includes name, size, progress, and status.
maxFileSize string '10 MB' Maximum allowed file size. Displayed in the hint text below the Browse button.
acceptedTypes string 'JPG, PNG, GIF, PDF' Comma-separated list of accepted file types. Displayed in the hint text.
multiple boolean true Allows multiple files to be selected or dropped simultaneously.
showAddMore boolean true Shows the Add More button in uploading and complete states.
onFileDrop function — Callback fired when files are dropped onto the zone.
onBrowse function — Callback fired when the Browse Files button is clicked.
onRetryFailed function — Callback fired when the Retry Failed button is clicked in error state.

Accessibility

Keyboard Navigation

  • Tab moves focus to the Browse Files button and to each file row's action button (cancel or remove).
  • Enter / Space activates the Browse Files button, opening the file picker dialog.
  • Escape cancels an active drag operation and returns the zone to its default state.

Screen Reader

  • The drop zone uses role="region" with an aria-label (e.g. "File upload area") to give it a descriptive landmark.
  • Drag-over state changes are announced via aria-live="polite" on a visually hidden status element, so users know when the zone is ready to accept a drop.
  • Each file row's progress bar uses role="progressbar" with aria-valuenow, aria-valuemin="0", and aria-valuemax="100".

Status Announcements

  • Batch completion and error summaries ("3 files uploaded", "2 of 4 failed") use aria-live="polite" so they are announced automatically without interrupting the user.
  • The "do not close this window" warning during upload uses role="alert" to ensure it is read immediately when the upload begins.
