Switch

A Switch is a binary toggle that lets users turn a setting on or off. It comes in two sizes — Default (44 × 24) and Compact (36 × 20) — and can be paired with a label and description.

Variants

Default size (44 × 24):
  • Each state shows Off (left) and On (right) side by side
  • Hover — adds a subtle halo behind the thumb
  • Focus — 2px blue ring + 4px glow around the track
  • Disabled — muted track colours, no interaction
Compact size (36 × 20):
  • Use in dense layouts — table rows, sidebars, toolbars
  • 16px thumb instead of 20px, all states identical to Default

Accessibility

  • Role: Use role="switch" with aria-checked="true/false". If using a native <input type="checkbox">, add role="switch" to override the checkbox semantics.
  • Keyboard: Space toggles the switch on/off. Tab moves focus to the next element. The focus ring (2px blue + 4px glow) wraps the entire track for clear visibility.
  • Label: Every switch must have a visible label associated via <label for="...">. Description text is connected via aria-describedby.
  • Motion: The thumb slides between positions. Respect prefers-reduced-motion by disabling the slide transition when the user has opted out of motion.

Related Components

Checkbox
Use a Checkbox when the toggle controls a form field that is submitted. Use Switch for immediate on/off settings.
Radio Button
Use Radio Buttons when choosing between 3+ mutually exclusive options. Switch is for binary on/off only.
Segmented Control
Use a Segmented Control when toggling between 2–4 named views. Switch is for enabling/disabling a feature.
