Feedback

Toast

Toasts are temporary notifications that surface system feedback without interrupting the user's workflow. They appear briefly, communicate the outcome of an action, and dismiss automatically. Four purposes are available — Error, Success, Info, and Warning — each with a distinct colour and icon.

Accessibility

  • Live region: Render toasts inside a container with role="status" (or role="alert" for errors) and aria-live="polite" so screen readers announce them without stealing focus.
  • Keyboard: The close button must be reachable via Tab. Pressing Esc while a toast is focused should dismiss it. Auto-dismiss timers must pause on hover and on focus.
  • Colour contrast: White text on the solid status colours meets WCAG AA. Verify in dark mode that the chosen status fills retain at least 4.5:1 contrast against the white text and icons.
  • Duration: Default 5 second auto-dismiss; allow users to pause or extend. For critical errors, use duration: 0 so the toast stays until manually dismissed.

Related Components

Alert
Use Alert for persistent inline messages tied to a specific page region. Toast is for transient, global feedback.
Badge
Use Badge for compact status labels embedded next to other content rather than as a notification.
Modal
Use Modal when feedback requires acknowledgement or a decision before the user proceeds.
