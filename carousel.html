Carousel

The Carousel component displays a series of images in a single framed slot with dot-based navigation and optional arrow controls. A position counter badge and fullscreen expand button can be overlaid on the image for added context and accessibility.

Carousel variants
  • Default — all controls shown: arrows, dots, badge, expand button
  • Minimal — dots only; no arrows, badge, or expand button; used in constrained layouts
  • No badge — hides the position counter; useful when the total count is implied
  • No expand — removes the fullscreen shortcut; use when a lightbox is not available
Navigation dots scale by proximity:
  • Active — 16px solid blue; marks the current image
  • Default — 12px outlined; adjacent images
  • Distant — 10px outlined; further images, creates depth gradient

Properties

Property Type Default Description
images string[] [] Array of image URLs to display in the carousel.
activeIndex number 0 Zero-based index of the currently displayed image.
showBadge boolean true Displays the image position counter (e.g. "1 / 6") overlaid at the bottom-left of the photo.
showButton boolean true Shows the previous and next arrow buttons flanking the navigation dots.
showExpand boolean true Shows the expand icon button at the bottom-right of the photo, typically triggering a lightbox.
onNext function — Callback fired when the next arrow or a dot to the right is activated.
onPrev function — Callback fired when the previous arrow or a dot to the left is activated.
onExpand function — Callback fired when the expand button is clicked.

Accessibility

Keyboard Navigation

  • Tab moves focus to the previous button, navigation dots, next button, and expand button in order.
  • Enter / Space activates the focused arrow button or dot.
  • Arrow Left / Arrow Right can be used within the dot group (when dots have role="tab") to navigate between images.

Screen Reader

  • The dot container uses role="tablist" with each dot as role="tab" and aria-label="Image N" so screen readers can announce position and navigate directly.
  • The active dot carries aria-selected="true"; inactive dots use aria-selected="false".
  • Arrow buttons use descriptive aria-label values ("Previous image", "Next image") since they contain only icons.
  • The photo <img> element must have a meaningful alt attribute describing the image content (e.g. "Tan leather sofa, front view").
  • The expand button uses aria-label="Expand image" and aria-haspopup="dialog" when it triggers a lightbox overlay.

Motion & Animation

  • Image transitions should respect prefers-reduced-motion — disable slide/fade animations when the user has requested reduced motion.
  • Auto-playing carousels must provide a pause control and must not advance faster than every 5 seconds per WCAG 2.2 guideline 2.2.2.
